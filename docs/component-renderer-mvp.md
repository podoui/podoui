# Component Renderer MVP

Phase 3 keeps component behavior shared and renderer output target-specific.

## Core Contract

- `@podo/core` owns component registry lookup, renderer contracts, accessibility wiring, and behavior helpers.
- Functional components compute state in behavior helpers first, then pass renderer-neutral output to each target renderer.
- Renderers do not parse component JSON directly during runtime; codegen and registry APIs are the boundary.

## Web Components

- MVP uses direct Custom Elements instead of Lit to keep runtime dependency size small.
- `registerPodoElements()` defines `podo-button`, `podo-input`, `podo-field`, `podo-icon`, and `podo-text`.
- Slots are native Web Component slots. Styling is CSS variable based and follows the token names emitted by `@podo/tokens`.

## React

- MVP uses native React renderers instead of Web Component wrappers.
- This keeps controlled/uncontrolled inputs, refs, event handlers, and Testing Library interactions idiomatic.
- `PodoThemeProvider` writes `data-podo-theme` and `data-color-scheme` for CSS variable scoping while exposing the same values through context.

## Hono

- Hono components are static TSX/SSR renderers.
- Events are emitted only as attributes or form state in Phase 3. Client interactivity is intentionally outside this MVP.
- `renderCriticalCss()` provides initial theme attributes and critical CSS insertion for Hono responses.
- Declarative Shadow DOM is not an official MVP support target. It requires a separate client-entry and hydration strategy before being added.

## React Native

- React Native renderers use a host adapter with `Pressable`, `Text`, `TextInput`, and `View`.
- The default host names match React Native primitives, while tests inject DOM-compatible hosts.
- `adaptReactNativeTokens()` converts web dimensions such as `px` and `rem` into React Native numbers.

## Codegen

- `@podo/codegen` generates deterministic target files from component specs.
- Every generated file starts with:

```ts
/* This file is generated by @podo/codegen. Do not edit directly. */
```

- Templates are separated by target in `componentTemplates`.
- Idempotency is tested by generating from differently ordered inputs and comparing the full output.
