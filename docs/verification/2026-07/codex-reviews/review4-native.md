## Blocking findings

1. `Icon` does not resolve names to native glyphs. It renders `props.glyph ?? props.name`, so `<Icon name="search" />` displays the literal word “search.” The harness explicitly observes that literal text and incorrectly calls it PASS. This contradicts the native glyph-map requirement and checked-in native example, which uses `<Icon name="menu" />` without `glyph`. The generated glyph map is never consumed by the component. [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:1163), [App.tsx](/Users/tarucy/project/podoui/examples/native/src/App.tsx:44), [icon-guide.md](/Users/tarucy/project/podoui/docs/icon-guide.md:92), `native.md` A5.

2. Native Button omits required interaction states. The component explicitly assumes native has no pressed state and always supplies a static style object, even though React Native `Pressable` exposes `pressed` specifically for visual feedback. The component spec declares an active/pressed state and lists no native limitation. Disabled rendering is also reduced to opacity over the original theme instead of the design system’s disabled fill/text treatment. A2 merely verifies the current `opacity: 0.56`; it never exercises press-in rendering. [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:408), [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:464), [button.component.json](/Users/tarucy/project/podoui/packages/spec/samples/components/button.component.json:210), [styles.css](/Users/tarucy/project/podoui/packages/podo-ui/styles.css:170), `native.md` A2.

3. The real React Native `ScrollView` path is incorrectly styled. Select passes padding, gap, and flex-direction through `ScrollView.style`; React Native requires child-layout styles on `contentContainerStyle`. The claimed 474px calculation depends on those padding/gap values, so the RNW measurement does not establish the real RN layout. The fake `TestScrollView` only reads `style.maxHeight` and cannot catch this. [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:931), [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:1728), [ScrollView.d.ts](/Users/tarucy/project/podoui/packages/native/node_modules/react-native/Libraries/Components/ScrollView/ScrollView.d.ts:605), [index.test.tsx](/Users/tarucy/project/podoui/packages/native/src/index.test.tsx:1070).

4. Long-list Select keyboard navigation does not keep the active option visible. Arrow keys only update `activeIndex`; the ScrollView has no ref and no `scrollTo`/visibility adjustment. After enough ArrowDown presses, `aria-activedescendant` points below the viewport while the visible menu provides no active feedback. B5 uses only three options, while B6 manually scrolls and mouse-clicks the 15-item list, avoiding this combined hard path. [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:709), [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:931), `native.md` B5–B6.

5. Field’s automatic character counter becomes stale after an external controlled-value update. `autoCount` is initialized once and changes only when the wrapped control invokes `onValueChange`; there is no synchronization when a parent replaces the child’s `value`. The harness proves only user typing, not controlled updates. [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:1080), [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:1589), harness `App.jsx:399-400`, `native.md` A11.

6. The claimed stale-press guard is not actually proven and is stale itself. `pick` captures `disabled`/`readOnly` from its render, so invoking an old queued handler after a lock transition still observes the old `false` values. The test and browser evidence click an already-detached DOM node; detached events do not reach React’s delegated root, so the zero callback count does not demonstrate the guard. [index.tsx](/Users/tarucy/project/podoui/packages/native/src/index.tsx:646), [index.test.tsx](/Users/tarucy/project/podoui/packages/native/src/index.test.tsx:743), `native.md` A8 (`staleCellStillConnected:false`).

7. There is still no valid checked-in real React Native example or real-RN execution evidence. Documentation says actual RN must inject hosts because top-level exports are unsupported, but `examples/native` imports those top-level exports and does not depend on `react-native`. The report’s honesty section also concedes that native gestures and focus order were not exercised. Thus RNW success cannot support the unrestricted claim about the native runtime.
